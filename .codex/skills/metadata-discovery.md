# Skill: metadata-discovery

## Goal

Inspect relevant Salesforce metadata using MCP and repository files.

## Rules

- Never guess Salesforce metadata names.
- Inspect only metadata relevant to the ticket.
- Do not modify files.
- Do not deploy.

## Inspect when relevant

- Objects
- Fields
- Validation rules
- Flows
- Apex classes
- Triggers
- Lightning Web Components
- Permission sets
- Profiles
- Layouts
- Flexipages
- Existing tests

## Output

- Existing components found
- Reusable components
- Potential conflicts
- Components likely to create
- Components likely to modify
- Discovery gaps
