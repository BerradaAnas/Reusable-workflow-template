# Skill: deployment-docs

## Goal

Create clear deployment, release, rollback, and validation documentation.

## Include

- Pre-deployment checklist
- Components to deploy
- Deployment steps
- Post-deployment validation
- Rollback steps
- Release notes
- Known risks
- UAT notes

## Rules

- For MVP projects, default to dev org only.
- Do not assume production exists.
- Identify manual approval gates.
- Keep rollback steps practical and specific.

## Output

- Deployment checklist
- Rollback plan
- Release summary
- Post-deployment validation steps
